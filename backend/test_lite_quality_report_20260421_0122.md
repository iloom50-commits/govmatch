- `ui` 마크다운 형식을 일관되게 사용하고 리스트의 가독성을 높이기 위해 적절한 공백과 구분선을 추가할 것.
- `ux` 이전 대화 내용을 더 잘 반영하고 사용자의 이해 수준을 감지하여 맞춤형 답변을 제공할 것.
- `quality` 정보의 정확성을 높이고 질문에 대한 답변의 완결성을 강화할 것.
- `ux` 사용자의 질문 의도를 더 잘 파악하고 이전 대화 맥락을 반영하여 답변을 조정할 것.

### [MID]
- `ui` 정보를 더 간결하게 정리하고, 리스트 형식을 일관되게 유지.
- `ui` 각 답변 말미에 사용자가 선택할 수 있는 몇 가지 후속 질문 또는 탐색 옵션(예: "취업 지원에 대해 더 알아볼까요?", "다른 분야의 지원금도 찾아드릴까요?")을 제시하여 대화의 주도권을 확보해야 합니다.
- `quality` "대출이 아닌 직접적인 지원금을 찾는 것이 어려운 상황"과 같은 부정적인 뉘앙스의 문구 대신, 현재 제공 가능한 긍정적인 대안을 강조하는 방식으로 답변을 개선해야 합니다.
- `quality` 지원금 정보의 출처와 조건을 명확히 제시하기.
- `ux` 사용자의 반응에 따라 후속 질문을 더 유연하게 조정하기.
- `quality` 중복된 정보 제공을 줄이고, 각 지원사업의 세부 정보를 더 명확하게 제공할 것.

### [LOW]
- `ui` 마크다운 사용 시, 각 지원사업의 핵심 정보(예: 지원금액, 대상)를 더 시각적으로 강조하거나 요약하는 방식을 고려해야 합니다.
- `quality` 대출 외의 지원금에 대한 정보를 더 다양하게 제공하기.
- `ui` 리스트 형식을 개선하여 가독성을 높일 것.
