# 에버그린 랜딩 (파일럿) — 소상공인 정책자금

작성일: 2026-07-16
상태: 설계 승인됨(구두) → 사용자 스펙 검토 대기

## 배경 / 목표
govmatch.kr의 검색 자산은 현재 **휘발성 공고 상세페이지**뿐이고, 그중 800개가 thin-content로 자기-noindex 상태(GSC 실측). 신규·저권위 도메인이라 구글이 프로그램 생성 페이지를 잘 색인하지 않음.

→ **안 시드는(에버그린) 랜딩페이지**를 신설해, "소상공인 정책자금" 같은 **고의도 상시 검색어**로 유입을 만들고 앱(맞춤 매칭)으로 전환한다. 이 파일럿 1장으로 **템플릿·색인·순위를 검증**한 뒤 같은 틀로 확장한다.

## 스코프 (이번 파일럿)
- 세그먼트: **기업·소상공인**
- 페이지 수: **1장** — 대표 키워드 "소상공인정책자금"
- 구조: **에버그린 가이드 + 라이브 공고 하이브리드**

## 페이지 구조 (모두 SSR — 크롤러가 본문 전체를 읽음)
1. **Breadcrumb**: 홈 › 가이드 › 소상공인 정책자금
2. **H1**: "소상공인 정책자금 총정리 — 자격·한도·금리·신청방법 (2026)"
3. 리드 문단 + 목차(앵커 링크)
4. **에버그린 가이드 본문**(안 만료, 대표 검수) — 소제목은 실측 연관어 기반:
   - 정책자금이란 / **일반 대출과 차이**(연관: 소상공인대출 31,770)
   - **종류·집행기관**: 소진공(직접대출·대리대출), 지역신용보증재단 보증부, 지자체 이차보전 — 여기서 **중진공(중소기업진흥공단)·중소기업 정책자금도 간략 비교**(중진공/중소기업 검색 흡수)
   - **자격요건**: 소상공인 기준·업종·기간 + **개인사업자정책자금**(18,340) 관점
   - 한도·**금리**(연도 표기, 변동 주의 문구)
   - **신청 절차·필요서류**(연관: 소상공인정책자금 신청 990)
   - 흔한 실수 / 탈락 사유 / 관련: **폐업지원금**(15,200) 링크
   - **FAQ**(5문항 내외 — 직접대출 vs 대리대출, 금리, 온라인 신청 등 연관어 반영)
5. **라이브 공고 섹션**: "지금 신청 가능한 정책자금 공고" — 기존 `public` API 필터 결과 N건(6~10건). 각 항목 → `/announcements/{id}` 내부링크(= 관련 공고 크롤 유도).
6. **CTA 배너**: "내 조건 맞춤 정책자금 30초 매칭" → 앱 진입(기존 매칭 플로우).
7. **JSON-LD**: `FAQPage` + `BreadcrumbList`.

## 데이터 흐름
- 서버컴포넌트가 요청 시(ISR `revalidate: 3600`):
  - (a) 가이드 본문 = slug 매핑에서 로드(정적, 앱 번들 내).
  - (b) 라이브 공고 = `GET {API}/api/announcements/public?target_type=business&size=10&{정책자금 필터}` 서버사이드 fetch. 백엔드 장애 시 공고 섹션만 숨기고 가이드는 정상 렌더(가이드가 페이지 본체라 thin-content 아님).
- **정책자금 필터 방식은 구현 시 확정**: 실제 `category` 값 분포를 확인해 `category=정책자금`(정확) vs `search=정책자금`(포괄) 중 선택. 둘 다 기존 API 파라미터라 백엔드 무변경.

## 메타데이터 / SEO
- `generateMetadata`: 고유 title("소상공인 정책자금 총정리 — 자격·한도·신청방법 | 지원금AI"), description(150자), `alternates.canonical`, openGraph(type article). **noindex 아님**(가이드 본문이 충분히 두꺼움 → thin 가드 미적용).
- 루트 layout의 title.template과 중복되지 않게 title은 접미사 없이 지정하거나 `absolute` 사용(직전 제목중복 이슈 참조).

## 테스트 / 검증
- `npm run build` 통과.
- 로컬 렌더: H1·가이드 본문·FAQ·라이브 공고·CTA가 **SSR HTML에 실제 텍스트로** 존재(로그인 무관, 스켈레톤 아님) — `curl -A Googlebot` 으로 본문 문자열 확인.
- JSON-LD 유효성(FAQPage/BreadcrumbList 파싱).
- 배포 후 GSC URL 검사 → **색인 생성 허용 = 예**, noindex 아님, canonical 일치.

## 성공 기준
- 배포 후 페이지가 **색인 대상**(noindex 아님)으로 크롤됨.
- 수 주 내 "소상공인 정책자금" 검색에 노출 시작(초기 하위 → 권위 축적하며 상승).
- 라이브 공고 내부링크로 해당 공고 크롤 빈도 개선.
- 검증되면 동일 템플릿으로 "창업지원금" 등 확장.
