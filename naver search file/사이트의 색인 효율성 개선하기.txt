사이트의 색인 효율성 개선하기
네이버 검색로봇으로 여러분의 웹 콘텐츠가 수집이 되어 검색엔진에게 전달이 되었다면 해당 문서는 색인 후보에 포함됩니다. 하지만 모든 수집된 문서가 색인되지 않으며 아래와 같이 색인에 불필요하다고 판단되는 불용문서는 색인 대상에서 제외됩니다. 아래는 불용문서의 대표적인 케이스를 예로 들었는데요 색인 효율성을 개선하기 위하여 사이트 개선을 통해서 해결하시길 바랍니다.

HTML 내용이 없는 경우
일반적인 웹페이지의 구성은 HTML 마크업, CSS 및 Image 리소스, 자바스크립트로 구성되어 있습니다. 네이버 검색로봇은 웹페이지의 콘텐츠를 해석할때 최신 웹 브라우저와 최대한 동일한 모습(렌더링)으로 해석하여 처리합니다.

특정 웹페이지의 경우 HTML 마크업이 없이 순수 자바스크립트를 활용하여 웹 페이지를 이동하는 경우가 있습니다. 이 경우 네이버 검색로봇이 제대로 해석할 수 없는 경우가 발생할 수 있으며 내용이 없는 웹페이지로 간주하여 색인에서 제외하는 경우가 발생할 수 있습니다. 웹 페이지 이동 가이드 에 언급한 바와 같이 되도록 자바스크립트가 아닌 HTTP 응답코드중 301 혹은 302 방식으로 이동을 권장합니다.

소프트 404 (혹은 소프트 데드)
사이트 구축 및 운영을 할때 오류 페이지의 경우 올바른 HTTP 응답코드를 유지하는 것이 매우 중요합니다. 예를들어서 아래와 같이 사이트의 예외적인 오류 상황이 발생하는 경우 사용자에게 적절한 안내 페이지를 제공함과 동시에 각 케이스별 HTTP 응답코드를 지정해야 합니다.

자료 없음, URL 잘못 입력, 옛날 URL : 404 (Not Found)
로그인 사용자만 사용할 수 있는 페이지 : 403 (Forbidden)
사이트가 다른 도메인으로 이전 : 301 (Moved Permanently)
사이트를 운영하면서 아래와 같은 상황도 발생할 수 있는데요, 아래의 경우에도 각 케이스별 적절한 HTTP 응답코드를 지정해야 합니다.

사이트 점검 : 503 (Service Unavailable)
웹 호스팅 기한이 만료되어 해당 사이트가 더 이상 운영되지 않는 경우 : 503 (Service Unavailable)
만약 위와 같은 오류 상황에서 HTTP 응답코드 200(OK)를 지정한다면 검색로봇은 정상적인 웹 콘텐츠로 인식하여 수집 후 색인대상으로 전달하게 됩니다. 색인 과정에서 불용문서의 케이스 검출된다면 자연스럽게 색인에서 제외되지만 간혹 불용문서로 인식을 못해 검색에 노출되는 경우가 발생되며, 해당 콘텐츠는 검색 사용자에게 아무런 도움이 되지 않는 문서이기 때문에 사이트의 신뢰성에 좋지 않는 영향을 줄 수 있습니다.
