# 크레딧 선불 과금 모델 전환 설계

> 작성일: 2026-07-12
> 상태: **미채택(보류)** — 2026-07-12 대표 결정으로 기존 구독 결제 유지, 본 크레딧 모델은 구현하지 않음. 향후 재검토용 설계 기록으로만 보존.
> 대상: 일반 사용자 과금 구조 (PRO 구독은 유지)

---

## 1. 배경 · 목표

### 문제
LITE/PRO **월정액 구독** 모델을 운영했으나 **구독자 0명**. 대표 진단:
> "한국 소비자는 국내 서비스에 대해 **구독료 지불 자체를 매우 꺼린다.**"

즉 실패 원인은 **가치·가격이 아니라 '정기결제 약정 구조'** 다. 가치를 못 느껴서가 아니라, "매달 자동으로 빠져나가는 약정"에 심리적 저항이 있다.

### 목표
일반 사용자 티어에서 **월정액 약정을 제거**하고, **필요할 때만 지불하는 크레딧 선불 충전·소모 모델**로 전환한다. PRO(전문가 구독)는 지속성 기능이라 그대로 유지한다.

### 성공 기준
- 비구독 상태에서도 유료 AI 기능을 **소액·무약정**으로 사용할 수 있다.
- "이 기능 누르면 얼마 나가는지"를 사용자가 **항상 사전에** 안다(투명성).
- 마중물 크레딧으로 결제 전 가치를 체험한다.

## 2. 왜 지금 '크레딧'인가 — 과거 횟수 과금 기각의 극복

`docs/PRICING_MODEL.md` 9절은 과거 **횟수 기반 과금을 기각**한 기록이다. 그 4가지 우려를 크레딧 선불 모델이 구조적으로 회피하도록 설계한다. 이 표가 본 설계의 핵심 원칙이다.

| 과거 기각 이유 | 크레딧 선불 모델의 해법 |
|---|---|
| ① 창만 열어도 1회 차감(불공정) | **세션 시작 시에만** 차감. 창을 열거나 목록을 보는 것은 무료. 기존 `consult_sessions` dedup 재사용해 24시간 내 후속 질문 무료 |
| ② "횟수가 줄어든다" 심리 위축 | 잔액이 **원(₩) 단위**라 "돈을 쓴다"는 인식이 공정·직관적. 마중물 + 월 무료로 완충 |
| ③ 안 쓰면 손해 → 다음달 이탈 | **크레딧 무기한 유효**(월 리셋 없음). "이번 달 안 쓰면 손해"라는 압박 자체가 없음. 정기결제가 아니라 취소 결정도 없음 |
| ④ 남은 소비량에 집착 | **투명 단가**(공고상담 50원 / 자금상담 100원)로 '가치=가격'이 직결. 소비량이 아니라 실제 사용 가치에 주목 |

과거엔 이 우려 때문에 "기능 등급제 구독"으로 갔고, 그 구독이 다시 **한국형 구독 기피**에 막혔다. 크레딧 선불은 등급제의 명료함(무엇에 얼마)을 유지하면서 **약정·월리셋을 제거**해 양쪽 문제를 동시에 푼다.

## 3. 모델 정의

### 3.1 티어 구조 (전환 후)

| 구분 | 비회원 | 일반 사용자 | PRO (49,000원/월) |
|---|---|---|---|
| 대상 | 둘러보기 | 모든 개인·사업자 | 컨설턴트/전문가 |
| 공고 검색·상세 | O | O | O |
| 맞춤 알림·저장 | X | O | O |
| 공고 상담 | X | **50크레딧/건** | 무제한(구독 포함) |
| 자금 상담 | X | **100크레딧/건** | 무제한(구독 포함) |
| 전문가 기능(고객사관리·리포트·화이트라벨) | X | X | 무제한(구독) |

- **FREE / LITE 개념 삭제.** 일반 사용자는 플랜 라벨 없이 "크레딧 잔액"만 갖는다. 플랜 선택·약정 화면 없음. 진입 문구는 "무료로 시작, 다 쓰면 충전" 한 줄.
- **PRO는 월 49,000원 구독 유지** + 상담 크레딧 **무제한 포함**(PRO 구독자는 크레딧 차감 없음).

### 3.2 크레딧 단위 · 소모표

- **1크레딧 = 1원** (선불 충전금. 이름만 '크레딧', 실질은 원 단위 선불금)
- 소모(세션 단위, 24h 내 후속 질문 무료):

| 기능 | 엔드포인트 | 소모 |
|---|---|---|
| 공고 상담(특정 공고 "나도 받을 수 있나?") | `POST /api/ai/consult` | **50크레딧** |
| 자금 상담(자금상담AI, 일반 상담) | `POST /api/ai/chat` | **100크레딧** |

> 매핑 확인 필요: 공고상담=`/api/ai/consult`, 자금상담=`/api/ai/chat`. 코드 조사 근거(Explore) 기준이며, 구현 전 실제 프론트 버튼→엔드포인트 최종 확인.

### 3.3 무료 크레딧 (마중물 + 월 무료)

- **가입 마중물: 300크레딧** (1회성) — 자금상담 3번 또는 공고상담 6번 체험 가능. 기존 "FREE 월3회" 대체.
- **월 무료: 매월 50크레딧** — 활동 시 지연 지급(lazy). 사용자가 행동하는 시점에 `monthly_free_reset_at`의 월이 바뀌었으면 +50 지급 후 리셋. 비활성 사용자에게 누적 지급되지 않음(월 최대 50). 재방문 습관 루프 유지 목적.

### 3.4 충전 (선불)

- **최소 충전 5,000원** (1천원 단위 소액은 PG 고정수수료가 마진 잠식 → 하한).
- 패키지(보너스로 재충전 유도):

| 충전액 | 지급 크레딧 | 보너스 |
|---|---|---|
| 5,000원 | 5,000 | — |
| 10,000원 | 11,000 | +10% |
| 30,000원 | 34,500 | +15% |

- 보너스 크레딧은 원장에 별도 사유(`topup_bonus`)로 적립. 환불 시 보너스분은 제외(9절).

### 3.5 유효기간

- **초기 무기한.** 환불 민원·신뢰 이슈 회피 + "안 쓰면 손해" 압박 제거(2절 ③). 악용 징후가 보이면 이후 정책 도입(예: 최종 사용일로부터 N년).

## 4. 설명 · 투명성 UX (핵심 요구사항)

새 과금 모델의 성패는 **"사용자가 비용을 항상 사전에 이해하는가"** 에 달렸다. 아래 8개는 설계 필수 요소다.

1. **잔액 상시 노출** — 헤더/사이드바에 "보유 크레딧 1,250" 항상 표시. 클릭 → 크레딧 안내 모달.
2. **기능별 소모량 배지** — 버튼에 비용 명시: "공고 상담 · 50크레딧", "자금 상담 · 100크레딧". 누르기 전에 안다.
3. **소모 전 확인** — 세션 시작 시 1회 확인: "이 상담은 50크레딧이 차감됩니다. 후속 질문은 24시간 무료예요. [시작]". 창만 열면 차감 안 됨을 명시.
4. **잔액 부족 안내** — "크레딧이 부족해요 (현재 20 / 필요 50)" → [충전하기] → 충전 모달.
5. **크레딧 안내 페이지/모달** — 한눈에: 1크레딧=1원, 소모표, 마중물/월무료, 무기한 유효, 충전 패키지·보너스, 환불정책.
6. **차감 후 피드백** — "50크레딧 사용 · 잔액 1,200" 토스트.
7. **거래 내역(원장)** — 마이페이지에 충전/사용/보너스/마중물 이력(일시·사유·증감·잔액). 투명성 = 신뢰.
8. **온보딩** — 가입 직후 1회: "환영합니다! 마중물 300크레딧을 드렸어요. 자금상담 3번 or 공고상담 6번 무료 체험하세요."

## 5. 데이터 모델 (DB — ADD only)

배포 정책상 **컬럼 삭제/변경 금지, 추가만 허용**. 기존 plan 관련 컬럼은 보존한다.

### 5.1 `users` 추가 컬럼
- `credit_balance INTEGER DEFAULT 0` — 현재 크레딧 잔액(원).
- `monthly_free_reset_at TIMESTAMP` — 월 무료 지급 기준 시각.

### 5.2 신규 테이블 `credit_ledger` (충전·소모 원장)
```sql
CREATE TABLE IF NOT EXISTS credit_ledger (
  id            BIGSERIAL PRIMARY KEY,
  user_id       INTEGER NOT NULL,
  delta         INTEGER NOT NULL,          -- +충전/보너스/마중물/월무료, -소모/-환불
  balance_after INTEGER NOT NULL,          -- 거래 후 잔액(감사·표시용)
  reason        VARCHAR(30) NOT NULL,      -- signup_grant|monthly_free|topup|topup_bonus|consult|fund_chat|refund|admin
  ref_id        TEXT,                      -- 결제 payment_id 또는 상담 session_id
  detail        TEXT,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_credit_ledger_user ON credit_ledger(user_id, created_at DESC);
```

- 잔액은 `users.credit_balance`가 정본(빠른 조회), `credit_ledger`는 감사·내역·복구용. 모든 증감은 **원장 기록 + 잔액 UPDATE를 한 트랜잭션**으로 처리.

## 6. 소모 / 차감 로직

차감 훅 지점은 기존 사용량 게이트 2곳을 재사용한다: `/api/ai/consult`(공고상담 50), `/api/ai/chat`(자금상담 100).

차감 규칙(공통 헬퍼 `_charge_credits(user, cost, reason, session_id)`):
1. **PRO 구독자(plan in pro/biz, 만료 전)** → 무제한. 차감 없음.
2. **기존 세션(24h 내 동일 `consult_sessions`)** → 후속 질문. 차감 없음(dedup 재사용).
3. **새 세션** → `credit_balance >= cost` 확인:
   - 충분: `credit_balance -= cost`, 원장 기록, 세션 생성.
   - 부족: **402** 반환 `{code:"INSUFFICIENT_CREDIT", need, have}` → 프론트 충전 유도.
4. 월 무료 지급은 소비 진입 시 lazy 체크: `monthly_free_reset_at` 월 변경 시 +50 지급 후 갱신.

기존 FREE 월3회 카운터(`ai_usage_month`) 로직은 **크레딧 차감으로 대체**(카운터 컬럼은 보존하되 게이트 판정에서 미사용).

## 7. 충전 결제 (PortOne 1회성)

- 기존 정기결제(빌링키)와 별개로 **1회성 결제** 사용. PortOne V2 `requestPayment`(카카오페이/카드) — 빌링키 발급이 아니라 단건 승인.
- 흐름: 프론트 충전 모달(패키지 선택) → PortOne 결제 → 서버 `POST /api/credits/topup`에서 **결제 검증(S2S)** → 검증 성공 시 `credit_balance += (액면 + 보너스)`, 원장 2줄(`topup`, 필요시 `topup_bonus`) 기록.
- 결제 검증은 기존 `_charge_billing_key`/PortOne 결제조회 API 패턴 재사용.
- 최소 5,000원 서버 측 검증(프론트 우회 방지).

## 8. 마이그레이션 · 기존 사용자

- **LITE 구독자: 0명** → 마이그레이션 이슈 없음.
- **기존 FREE 사용자** → 전환 시 **마중물 300크레딧 1회 지급**(원장 `signup_grant`). 불이익 없이 새 모델 진입.
- **PRO 구독자** → 변경 없음. 구독·자동갱신 유지 + 크레딧 무제한.
- 배포 후 `PRICING_MODEL.md`를 신 모델로 갱신(구 4단계 구독표 → 크레딧 모델).

## 9. 법적 · 정책

- **선불충전금 성격.** 1크레딧=1원 선불 충전금은 전자금융거래법상 선불전자지급수단에 해당할 수 있으나, **소규모 발행(발행잔액·거래규모 기준 이하)은 등록 면제** 대상. 규모 성장 시 재검토 항목으로 명시.
- **환불 정책(약관 명시 필요):**
  - 충전 원금(미사용분)은 사용자 요청 시 환불(PG 수수료 실비 공제 가능).
  - **보너스 크레딧은 환불 대상 제외**(무상 지급분).
  - 마중물·월무료 크레딧은 환불·현금화 불가.
- 크레딧 안내 페이지에 유효기간(무기한)·환불정책을 명시.

## 10. 범위 (In / Out)

**In scope**
- 일반 사용자 크레딧 충전·소모(공고상담 50 / 자금상담 100).
- 마중물 300 · 월무료 50 · 충전 패키지·보너스.
- 잔액/원장 DB, 차감 훅, 1회성 충전 결제, 설명·투명성 UX 8종.
- PRO 구독 유지(무제한 포함).

**Out of scope (후속 별건)**
- **SmartDoc 신청서 작성 건별 과금** — 엔진 미가동. 크레딧 인프라 위에 후속 설계(예: 신청서 1건 = N크레딧).
- 삼성페이 등 결제수단 확장.
- 크레딧 유효기간 도입, 선불수단 정식 등록(규모 성장 시).
- 기존 정기결제(빌링키) 코드 제거 — PRO가 계속 사용하므로 유지.

## 11. 오픈 이슈 (구현 전 확정)

1. 공고상담=`/api/ai/consult`, 자금상담=`/api/ai/chat` 매핑 실제 프론트 버튼 기준 최종 확인.
2. 월무료 50크레딧 지급을 "잔액에 가산" vs "잔액이 50 미만일 때만 채움" 중 택1(초안: 가산, 단 월 최대 50·lazy).
3. PRO 구독자에게도 크레딧 잔액 UI를 노출할지(초안: 노출하되 "구독 중 무제한" 배지로 대체).
4. 충전 패키지 3종(5천/1만/3만) 외 자유 금액 입력 허용 여부(초안: 패키지 고정, 추후 확장).
