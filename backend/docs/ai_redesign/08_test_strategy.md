# 08 테스트 전략 — 페르소나별 품질 검증

**목적**: 각 AI의 미션 달성 여부를 **객관적·반복적**으로 측정.

---

## 1. 테스트 원칙

### 1.1 미션 기반 페르소나 설계
- 각 AI의 미션(00_overview §2.1)이 충족되는지 확인
- 페르소나는 **실제 사용자 분포**를 반영 (연령·업종·상황)
- **경계 케이스 포함** (애매한 자격, 흐릿한 요청, 주제 전환)

### 1.2 교차 평가 (Gemini + OpenAI)
- 두 AI에게 동일 대화 보여주고 0~10점 매기게 함
- 평균 점수 + 차이 분석
- 한 AI가 후하거나 박하면 다른 쪽 보정

### 1.3 Variance 완화
- **각 페르소나 × 3회 반복 실행** → 평균
- Temperature 고정 (0.3~0.7 합의)
- Seed 사용 가능하면 고정

### 1.4 정량 + 정성 지표
- **정량**: Schema 준수율, 도구 호출 성공률, 응답 시간
- **정성**: 답변 품질 점수, 사용자 관점 만족도

## 3. 평가 Rubric (각 AI 공통)

### 3.1 정량 (자동 측정)

| 항목 | 측정 |
|---|---|
| Schema 준수율 | response_schema 위반 여부 |
| 필수 필드 채움률 | extracted_info, verdict, citations 등 |
| 응답 시간 | 호출당 평균 초 |
| 도구 호출 성공률 | tool call 에러 발생 비율 |
| 범위 밖 응대 일관성 | "저는 ... 전문 상담사입니다" 매칭율 |

### 3.2 정성 (AI 평가)

각 대화에 대해 Gemini + OpenAI가 0~10점 평가:

```
평가 기준 (각 0~10점):
1. 미션 달성: AI의 미션이 충족됐는가
2. 정확성: 근거 기반인가, 추측 없는가
3. 완결성: 질문에 충분한 답을 주었는가
4. 적절성: 사용자 프로필에 맞는가
5. 구조화: 답변이 읽기 쉬운가 (포맷)
```

**최종 점수**: 평균 × (Schema 준수율)

## 4. 자동 테스트 스크립트

### 4.1 구조

```
backend/tests/ai_redesign/
├── test_lite_announce.py      (페르소나 1~4)
├── test_lite_fund.py          (페르소나 1~4)
├── test_pro_match.py          (페르소나 1~5)
├── test_pro_announce.py       (페르소나 1~4)
├── test_reports.py            (LITE 2 + PRO 2)
└── common/
    ├── personas.py            (페르소나 정의)
    ├── evaluator.py           (Gemini + OpenAI 교차 평가)
    └── report_generator.py    (결과 보고서 MD 생성)
```

### 4.2 실행

```bash
# 전체 테스트
python -m backend.tests.ai_redesign.run_all

# 특정 AI만
python -m backend.tests.ai_redesign.test_lite_announce

# Variance 완화 (각 페르소나 3회)
python -m backend.tests.ai_redesign.run_all --repeat 3
```

### 4.3 출력

- `test_report_YYYYMMDD_HHMM.md` — 전체 결과
- `test_raw_YYYYMMDD_HHMM.json` — 원본 대화 + 평가

## 6. 합격 기준 (Go/No-Go)

### 6.1 Go 조건 (재설계 성공)
- 평균 점수 7.0 이상
- 치명적 버그(B1~B6) 모두 해결
- Schema 준수율 95% 이상
- 범위 밖 응대 일관성 100%

### 6.2 No-Go 조건 (재설계 재검토)
- 평균 점수 6.0 미만
- 특정 페르소나에서 지속 저점 (5.0 미만)
- Schema 위반 빈발 (5% 초과)

## 7. 회귀 테스트

### 7.1 매 배포 전
- 전체 페르소나 × 1회 실행 (15분)
- 점수 하락 > 0.5 시 배포 보류

### 7.2 매주 자동 실행
- 전체 페르소나 × 3회 평균
- Slack/이메일 알림 (점수·variance 추이)

### 7.3 A/B 테스트
- 신규 AI 활성화 비율을 10%→50%→100%로 확대
- 각 단계에서 점수·사용자 피드백 확인
